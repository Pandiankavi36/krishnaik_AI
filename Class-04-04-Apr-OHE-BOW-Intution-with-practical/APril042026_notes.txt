1) Easy to implement

2) captures word frequency

more occurrences - higher importance

3) works well before embedding for classical NLP tasks
useful for 
Text classification
Spam detection
Sentiment analysis (basic level)

4) NO training required
No learning Phase
Direct Conversion from text - numbers 
========================================================
CONS:

1) Ignores word orders
Sentence meaning can change, but BOW won't notice

exmpale;
"dog bites man"
man bites dog
Same BOW representation
BOW vector { dog, bites, man}
[1,1,1]
[1,1,1,]same vector


2) No Semantic understanding

Similar words are treated completely different
expl:
like vs love
 No relation captured
3) High dimensionality

vocabulary size = vector size  vocabulary - 50K the vector size = 50K 
Large dataset => huge vectors

4) Sparse Representation 
Mostly values are 0
Memory inefficient
Computation waste

5) OOV(out of Vocabulary) problem
New words cannot be handled
if word not in vocabulary - ignored or error 

6) Overemphasized frequent words
 Repeated words my dominate 
Example; "movie movie  movie"    -- There is no meaning for this sentence 

Gets high importance even if not meaningful
